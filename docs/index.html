üöÄ Quick Start

Get up and running with MUNO in minutes. Transform your multi-repository chaos into an organized, navigable workspace.

Installation

# Clone and build MUNO
git clone https://github.com/taokim/muno.git
cd muno
make build
sudo make install

Your First Workspace

# Initialize a new workspace
muno init my-platform
cd my-platform

# Add your repositories as a tree structure
muno add https://github.com/org/backend-team --name team-backend
muno add https://github.com/org/frontend-team --name team-frontend

# Navigate into backend team
muno use team-backend

# Add child repositories
muno add https://github.com/org/payment-service
muno add https://github.com/org/order-service
muno add https://github.com/org/shared-libs --lazy # Won't clone until needed

Navigate Your Tree

# View your repository tree
muno tree

# Navigate like a filesystem
muno use payment-service # Go deeper
muno use .. # Go up one level
muno use / # Go to root

# Work with Git at any level
muno pull --recursive # Pull entire subtree
muno status # Check current node status

‚ú® Why MUNO?

  • Tree-Based Navigation: Navigate repositories like a filesystem - intuitive and powerful
  • Lazy Loading: Repositories clone only when you need them, saving time and space
  • CWD-First Resolution: Your current directory determines the operation scope
  • Clear Targeting: Every command shows exactly what it will affect
  • Native Git Operations: All Git commands work naturally at any tree level
  • AI-Powered Sessions: Launch Claude Code sessions at any point in your tree
  • Single Binary: No runtime dependencies - just one fast Go binary
